🎬Customer Segmenentation Analytics - Summary
---
E-commercecn customer segmentation using clustering methods on e‑commerce customer data.
The project applies data exploration, feature preparation, clustering, and visualisation techniques, with emphasis placed on understanding results, model behaviour, and practical considerations rather than optimisation.

✨ Technologies
---
Python (google colab) - numpy, pandas, seaborn, matlplotlib, sklearn
The analysis was carried out using Python within a Jupyter Notebook environment, supported by commonly used data analysis and machine learning libraries.


🚀 Approach
---
The data was imported and exploratory data analysis was performed, including review of data quality, missing values, duplicates, and feature distributions. New features were created to convert transaction‑level data into customer‑level metrics to support segmentation.
Multiple clustering approaches were then applied to identify suitable customer groupings. Different methods were used to assess the optimal number of clusters, followed by clustering and visualisation to support interpretation. Throughout the analysis, observations were reviewed alongside assumptions, subjectivity, and limitations in the methods applied.


🚦 Running the Project
---
ipynb file can be run directly on google colab

The project can be run locally by cloning the repository and executing the notebook in a Python environment with the required libraries installed. The notebook is structured sequentially and can be run end‑to‑end.


🎞️ Preview of the output 
--- 
The outputs include analytical summaries and visualisations that show how customers are grouped under different clustering approaches. These outputs are used to support interpretation of segments and discussion of their practical relevance.


🔭 Why this project? 
--- 
This project was undertaken to gain hands‑on experience with data science and machine learning techniques, with a specific focus on understanding how models behave in practice, including assumptions, subjectivity, and limitations.
The work supports a broader objective of strengthening practical understanding of ML and AI concepts that are increasingly relevant in technology risk, governance, and assurance contexts, where interpretability and appropriate use are key considerations.
